Introducción

Los sistemas numéricos constituyen la base invisible pero vital de toda la infraestructura digital. Autores contemporáneos como Rosen (2020) y Stallings (2021) subrayan que el dominio de las bases binaria, octal y hexadecimal es una necesidad crítica para optimizar el almacenamiento de datos y la velocidad de procesamiento de datos.

El sistema binario se asocia con sistemas de señales discretos, mientras que el hexadecimal facilita la gestión de direcciones de memoria y protocolos de red avanzados como IPv6. La habilidad para convertir numeros entre diferentes sistmas numericos es fundamental para el diseño de hardware y el procesamiento digital de señales.

⚡ Procesamiento

La lógica binaria, octal y hexadecimal define la arquitectura y direccionamiento en el CPU y memoria física.

🔒 Seguridad

El manejo de bases de datos binarias y de cifrado hexadecimal es la base de la criptografía moderna.

1. Sistema Maya (Vigesimal)

Sistema posicional vertical basado en potencias de 20. Utiliza el punto (1), la barra (5) y el caracol (0).

2. Sistema Romano (No Posicional)

Basado en reglas aditivas y sustractivas con símbolos fijos (I, V, X, L, C, D, M).

3. Decimal a Binario

El sistema binario es posicional y tiene como base el 2, se representa unicamente dos digitos (0 y 1)

Para realizar conversiones se utiliza el algoritmo de divisiones sucesivas entre un numero decimal y el 2 para obtener la representación binaria

4. Sistema Octal (Base 8)

El sistema octal es posicional utiliza como base el numero 8, del 0 al 7. Es utilizado en computación ya que permite empaquetar de forma directa cadenas de 3 bits binarios.

Para realizar conversiones se utiliza el algoritmo de divisiones sucesivas entre un numero decimal y el 8, el residuo representa el valor octal

5. Sistema Hexadecimal (Base 16)

El sistema de base 16 utiliza los caracteres A, B, C, D, E y F para representar los valores del 10 al 15 respectivamente. Cada dígito hexadecimal se corresponde de manera exacta con una cadena de 4 bits binarios (nibble).

Para realizar conversiones se utiliza el algoritmo de divisiones sucesivas entre un numero decimal y 16 para obtener la representación hexadecimal
